## Request/Response Flow

### Online Mode (Network Available)

```
User Action
    ↓
Browser Request
    ↓
Service Worker Intercepts
    ↓
Check Strategy (Network First / Cache First)
    ↓
For Network First:
  ├─ Try Network → Success → Cache it → Return response
  └─ Network fails → Return from cache → (Fallback)
    ↓
For Cache First:
  ├─ Check cache → Found → Return cached response
  └─ Cache miss → Fetch from network → Cache it → Return
    ↓
Display to User
```

### Offline Mode (No Network)

```
User Action (Offline)
    ↓
Browser Request
    ↓
Service Worker Intercepts
    ↓
Network Check: ❌ OFFLINE
    ↓
Return from Cache
    ├─ If cached → Display cached response ✅
    └─ If not cached → Show offline.html ⚠️
    ↓
User sees offline content
```

## Installation Flow

### Step 1: Detection

```
Page Loads → beforeinstallprompt event → Stored in React state
```

### Step 2: Display Prompt

```
PWAInstallPrompt Component Renders
    ├─ Checks if app already installed
    ├─ If installed: Don't show prompt
    └─ If not installed: Show beautiful gradient UI
```

### Step 3: User Action

```
User clicks "Install"
    ↓
deferredPrompt.prompt() executes
    ↓
Browser shows native install dialog
    ↓
User confirms
    ↓
appinstalled event fires
    ↓
App added to home screen ✅
```

### Step 4: Standalone Mode

```
User launches app from home screen
    ├─ App runs in standalone mode
    ├─ No browser UI (full screen)
    ├─ Service worker active
    └─ All PWA features available
```

## Notification Flow

### Requesting Permission

```
User Action (Request Notification)
    ↓
Notification.requestPermission()
    ↓
Browser Shows Permission Dialog
    ├─ User clicks "Allow" → Permission granted ✅
    └─ User clicks "Block" → Permission denied ❌
    ↓
Store permission in component state
    ↓
Show appropriate UI based on permission
```

### Sending Notification

```
sendNotification("Title", { options })
    ↓
Check if Notification permission === "granted"
    ↓
Yes:
  ├─ If Service Worker controller exists
  │   └─ Post message to SW
  │        └─ SW.showNotification()
  └─ Fallback: new Notification()
    ↓
Notification appears (even if tab closed)
```

## Component Integration Points

### In Your App

1. **Install Prompt** → Automatically shown via `PWAInstallPrompt` in layout
2. **Service Worker** → Automatically registered by next-pwa
3. **Caching** → Automatic via runtime caching config
4. **Offline** → Automatic via service worker

### Optional Integrations

```javascript
// In SOS page
import { sendNotification } from "@/utils/pwaUtils";
sendNotification("Emergency SOS", { /* options */ });

// In Chat page
import { setupPWA } from "@/utils/pwaUtils";
useEffect(() => setupPWA(), []);

// In Share page
import { shareApp } from "@/utils/pwaUtils";
onClick={() => shareApp()};

// For debugging
import PWATestPanel from "@/components/PWATestPanel";
// Include in admin/dev page
```
